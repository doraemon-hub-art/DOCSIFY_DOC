# 架构设计

## 流控与背压


| 术语 | 含义 | 方向 |
|------|------|------|
| **流控（Flow Control）** | 控制数据发送速率，防止接收方被淹没 | 发送方主动控制 |
| **背压（Backpressure）** | 接收方反馈压力，让发送方减速 | 接收方反向通知 |

两者本质是同一问题的两面——**生产速度 > 消费速度时怎么办**。

> 常见的背压策略

- 阻塞； 
- 丢弃；
- 降速；
- 滑动窗口；
- 背压传播

> 设计的权衡

```
                  不丢消息
                    │
                    │
  低延迟 ───────────┼─────────── 高吞吐
                    │
                    │
                  低内存
```

三个目标不可兼得：

- **要低延迟 + 不丢消息** → 阻塞，可能 OOM(Out Of Memory 内存溢出)；
- **要高吞吐 + 不丢消息** → 磁盘缓冲，延迟增加；
- **要低延迟 + 高吞吐** → 丢消息，有损；
