# AI Agent 核心概念

---

## 0. 什么是 AI Agent？

### 一句话定义

**AI Agent（智能体）= 大模型 + 记忆 + 规划 + 工具使用**

它是一个能**自主理解目标、制定计划、调用工具、执行行动**的智能系统，而不仅仅是一个聊天机器人。

### 类比理解

```
大模型/Chat = 一个知识渊博的"顾问"
  → 你问它答，它不会主动做事

AI Agent   = 一个能自己干活的"员工"
  → 你给它一个目标，它自己想办法完成
```

### 具体对比

| 对比项 | 大模型 Chat（如 ChatGPT） | AI Agent |
|--------|--------------------------|----------|
| **工作方式** | 被动回答：你问一句，它答一句 | 主动执行：你给目标，它自己规划行动 |
| **记忆** | 单轮对话记忆，用完即忘 | 有长期记忆，能记住历史任务和结果 |
| **工具使用** | ❌ 不能（除非插件） | ✅ 能调用各种工具（Skill） |
| **自主决策** | ❌ 每一步都要你引导 | ✅ 能自己决定下一步做什么 |
| **任务闭环** | ❌ 只生成文字，不执行 | ✅ 从规划到执行到完成，全自动 |
| **典型例子** | 你问"什么是递归？"它解释 | 你让它"写一个递归程序并运行测试"，它自己完成 |

### 工作流程对比

```
大模型 Chat：
  用户输入 → 模型思考 → 文字输出 → 结束

AI Agent：
  用户给目标 → 模型思考 → 决定行动 → 调用工具 → 获取结果
       → 再次思考 → 再次行动 → ... → 目标完成 → 输出结果
```

### 一个具体例子

```
场景：帮我在闲鱼上找有趣的二手商品，生成视频创意

大模型 Chat：
  你："帮我找闲鱼上有趣的商品"
  Chat："抱歉，我无法访问闲鱼，建议你打开闲鱼APP查看"
  → 结束，它什么都做不了

AI Agent：
  你："帮我找闲鱼上有趣的商品，生成视频创意"
  Agent 开始工作：
    1. 思考："我需要先搜索闲鱼"
    2. 调用 Skill：search_xianyu("捡漏")
    3. 获取结果：[商品列表]
    4. 思考："这个 ThinkPad X230 不错"
    5. 调用 Skill：analyze_idea(商品信息)
    6. 生成创意："可以做'200元捡垃圾'主题视频"
    7. 调用 Skill：save_idea(创意内容)
    8. 完成 ✅
```

### 为什么需要 AI Agent？

```
大模型 Chat 的局限：
  - 只有"脑子"，没有"手"
  - 只能聊天，不能做事
  - 知识停留在训练时，没有实时信息

AI Agent 的突破：
  - 给大模型装上了"手"（工具/Skill）
  - 能自主规划、执行、反馈
  - 能获取实时信息，操作真实系统
```

### 总结

```
大模型 = 一个聪明但瘫痪在床的人（只会说，不会做）
AI Agent = 一个健全的员工（会想，也会做）

你的"两个AI员工"就是 AI Agent 的具体应用：
  - 它们有大模型作为"大脑"
  - 有 Skill 作为"工具"
  - 能自主完成任务，不需要你每一步都指挥
```

## 你正在使用的 AI Agent 例子

### Cursor（AI 驱动的 IDE）

**Cursor 是一个内置 AI Agent 的代码编辑器**，基于 VS Code 改造而来。

```
你平时怎么用 Cursor：
  1. 你提出需求："帮我写一个 Python 爬虫"
  2. Cursor 的 AI Agent 开始工作：
     ├── 理解你的需求（大模型思考）
     ├── 读取当前项目文件（Skill: read_file）
     ├── 生成代码（大模型生成）
     ├── 创建新文件（Skill: write_file）
     └── 给出运行建议（大模型输出）
  3. 你得到完整的代码文件 ✅

这正是一个 AI Agent 的典型工作流程：
  接收目标 → 规划 → 调用工具 → 完成
```

**Cursor 用到的 Skill（工具）：**

| Skill | 作用 |
|-------|------|
| `read_file` | 读取你项目中的文件，理解上下文 |
| `write_file` | 创建和修改代码文件 |
| `search_code` | 在项目中搜索代码 |
| `run_command` | 在终端执行命令 |
| `analyze_code` | 分析代码逻辑和错误 |

### VS Code Cline 插件

**Cline 是一个 VS Code 插件形式的 AI Agent**，和你当前对话的就是它。

```
你现在正在体验 AI Agent！
  1. 你提出想法："帮我整理 AI Agent 概念文档"
  2. Cline 开始工作：
     ├── 理解你的需求（大模型思考）
     ├── 读取 docs/AI/ 目录下的文件（Skill: list_files）
     ├── 读取现有 README.md（Skill: read_file）
     ├── 规划文档结构（大模型规划）
     ├── 创建 AIAgentConcept.md（Skill: write_to_file）
     ├── 根据你的反馈修改补充（Skill: replace_in_file）
     └── 完成文档 ✅
```

**Cline 用到的 Skill（工具）：**

| Skill | 作用 | 你看到的体现 |
|-------|------|-------------|
| `read_file` | 读取文件内容 | 我读取你的文档了解现有内容 |
| `write_to_file` | 创建新文件 | 我创建了 AIAgentConcept.md |
| `replace_in_file` | 修改文件部分内容 | 我补充了 AI Agent 定义章节 |
| `list_files` | 列出目录文件 | 我查看 docs/AI/ 目录结构 |
| `execute_command` | 执行终端命令 | 我运行命令检查环境 |
| `search_files` | 搜索文件内容 | 我在项目中查找相关内容 |
| `ask_followup_question` | 向用户提问 | 我问你"想往哪个方向深入？" |

### Cursor vs Cline 对比

| 对比项 | Cursor | VS Code Cline |
|--------|--------|---------------|
| **形态** | 独立 IDE（编辑器） | VS Code 插件 |
| **模型** | 内置云端模型 | 可配置（支持 Ollama 本地模型） |
| **Skill 数量** | 固定几个内置工具 | 可扩展，可自定义工具 |
| **自主性** | 中（需要你确认操作） | 高（可配置自动执行） |
| **适用场景** | 日常编码 | 复杂任务、自动化工作流 |
| **本地部署** | ❌ 必须联网 | ✅ 可完全本地（Ollama） |

### 这对你的启发

```
你现在通过 Cline 和我对话，本身就是 AI Agent 的实践：

你（用户）→ 提出目标 → Cline（Agent）→ 调用工具 → 完成任务
                      ↓
                底层调用大模型（DeepSeek/Claude 等）
                      ↓
                返回结果给你

未来你的"两个AI员工"也是同样的模式：
  - 你给目标（"每天搜闲鱼，生成视频创意"）
  - Agent 自主规划、调用 Skill、完成任务
  - 你只需要查看结果
```

## 1. AI 模型（7B 级别）

| 项目 | 说明 |
|------|------|
| **本质** | AI 的"大脑"，负责思考、分析、生成 |
| **推荐** | Qwen2.5-7B（中文能力最强） |
| **部署** | Ollama 本地运行，免费 |
| **速度** | 40-60 tokens/s（RTX 5070） |
| **能力** | 对话、分析、写代码、生成创意 |
| **局限** | 不能操作电脑、不能访问网页、知识有截止日期 |

### 常见 7B 模型对比

| 模型 | 中文能力 | 推理能力 | 速度 |
|------|---------|---------|------|
| **Qwen2.5-7B** | ⭐最强 | 强 | 快 |
| **DeepSeek-V2-Lite-7B** | 强 | 强 | 快 |
| **GLM-4-9B** | 强 | 中 | 中 |
| **Llama-3.1-8B** | 中（英文强） | 强 | 快 |

## 2. Ollama

| 项目 | 说明 |
|------|------|
| **本质** | 本地模型运行引擎 |
| **类比** | 机器人的"大脑" |
| **作用** | 在本地运行 AI 模型，提供 API 接口 |
| **部署** | WSL2 里一行命令安装 |
| **调用** | `http://localhost:11434` |
| **能** | 运行模型、回答问题、分析文本、生成创意 |
| **不能** | 操作电脑、访问网页、点击鼠标 |

### 安装命令

```bash
# Linux/WSL2
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen2.5:7b

# 启动服务
ollama serve
```

## 3. Skill（技能）

| 项目 | 说明 |
|------|------|
| **本质** | 给 AI 的"能力接口"（就是一个函数） |
| **类比** | 机器人的"工具包" |
| **作用** | 让 AI 能做原本做不到的事 |
| **例子** | 搜索网页、读写文件、调用 API、操控浏览器 |
| **部署** | 作为 Python 函数，挂载到 Agent 上 |

### Skill 的核心三要素

```
Skill = 触发条件 + 执行逻辑 + 返回结果
```

### 常见 Skill 示例

| Skill 名称 | 功能 | 输入 | 输出 |
|-----------|------|------|------|
| `search_xianyu` | 搜索闲鱼商品 | 关键词、价格范围 | 商品列表 |
| `read_file` | 读取文件 | 文件路径 | 文件内容 |
| `write_file` | 写入文件 | 路径、内容 | 成功/失败 |
| `analyze_idea` | 分析创意价值 | 商品信息 | 视频创意方案 |
| `web_search` | 搜索网页 | 查询词 | 搜索结果 |

### Skill 在不同框架中的叫法

| 框架 | 叫法 |
|------|------|
| LangChain | Tool（工具） |
| CrewAI | Tool（工具） |
| AutoGPT | Command（命令） |
| OpenAI | Function Calling（函数调用） |
| 自定义 | Skill（技能） |

## 4. Open CLAW（爪子）

| 项目 | 说明 |
|------|------|
| **本质** | 让 AI 操控电脑的工具 |
| **类比** | 机器人的"手" |
| **作用** | 模拟人类操作电脑：移动鼠标、点击、键盘输入、读取屏幕 |
| **部署** | Windows 上（操控真实浏览器） |
| **能** | 打开浏览器、访问网页、点击按钮、输入文字、截图 |
| **不能** | 自己思考、分析内容、做决策 |

### CLAW 是一种特殊的 Skill

```
Skill（技能）
  ├── search_xianyu（搜索闲鱼）← 你自己写的 Skill
  ├── read_file（读文件）      ← 通用 Skill
  ├── write_file（写文件）     ← 通用 Skill
  └── CLAW（操控电脑）         ← 一种特殊的 Skill
       ├── 移动鼠标
       ├── 点击按钮
       ├── 键盘输入
       └── 读取屏幕
```

## 5. 三者关系

```
Ollama = 大脑（思考）
CLAW   = 手（执行）
Skill  = 能力接口（连接大脑和手）

工作流程：
  1. Agent 问 Ollama："搜什么？"
  2. Ollama 回答："搜'捡漏'"
  3. Agent 指挥 CLAW："打开闲鱼，搜索'捡漏'"
  4. CLAW 执行并返回结果
  5. Agent 再问 Ollama："这个能做视频吗？"
  6. Ollama 分析并生成创意
```

## 6. 部署方案

### 推荐：混合部署

```
Windows 端：
  ├── CLAW + 浏览器（操控闲鱼网页）
  └── 查看结果

WSL2 (Ubuntu) 端：
  ├── Ollama + Qwen2.5-7B（模型推理）
  ├── Agent 逻辑（任务编排）
  └── cron 定时任务（自动触发）
```

### 通信方式

```
Windows CLAW ←→ localhost → WSL2 Agent ←→ localhost → WSL2 Ollama
```

## 7. 硬件需求

| 配置 | 需求 | 说明 |
|------|------|------|
| 显卡 | 6GB+ 显存 | 跑 7B 模型 Q4 量化 |

| 系统 | Windows 11 | WSL2 GPU 支持完善 |

### RTX 5070 (12GB) 方案

```
同时跑两个 7B 模型 (Q4)：
  ├── 模型A: ~5.5GB
  ├── 模型B: ~5.5GB
  └── 剩余: ~1GB（够用）

或分时复用：
  白天：一个模型，两个 Agent 切换上下文
  晚上：两个模型，全量利用
```

## 8. 主流 AI Agent 框架

| 框架 | 语言 | 特点 | 适合场景 |
|------|------|------|---------|
| **CrewAI** | Python | 多 Agent 协作，角色分工清晰 | ⭐ 最适合两个员工场景 |
